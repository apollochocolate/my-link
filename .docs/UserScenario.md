# 🗺 마이링크 (MyLink) 사용자 시나리오 (User Scenario)

## 1. 방문자 (Visitor): 마이링크 페이지 방문 과정

- **접속 및 프로필 확인**
  - 방문자는 특정 사용자의 소셜 미디어나 포트폴리오를 보기 위해 `domain.com/사용자닉네임` 경로로 접속한다.
  - 방문자는 화면 상단의 프로필 영역(닉네임, 한 줄 소개)을 읽기 위해 시선을 이동한다.
- **링크 탐색 및 클릭**
  - 방문자는 자신이 원하는 목적지(예: 깃허브, 블로그)를 찾기 위해 파비콘 이미지와 제목이 적힌 링크 리스트를 살펴본다.
  - 방문자는 기존 마이링크 화면을 잃지 않고 목적지로 안전하게 이동하기 위해 해당 링크 버튼을 클릭하여 **새 탭(New Tab)**으로 웹사이트를 탐색한다.

---

## 2. 소유자 (Owner): 링크 관리 과정

### 2.1 링크 추가 (Create)
- 소유자는 내 프로필에 새로운 사이트를 홍보하기 위해 관리자 화면의 **[새 링크 추가]** 버튼을 누른다.
- 소유자는 목적지를 설정하기 위해 노출된 입력 폼에 제목과 URL을 작성한다. (이때 사용자가 편의를 위해 `https://`를 생략해도 **시스템이 자동으로 `https://`를 최앞단에 붙여주어** URL 규격을 맞춘다.)
- 소유자는 작성한 내용을 반영하기 위해 **[저장]** 버튼을 클릭하고, 목록에 즉시 렌더링 된 '구글 파비콘 아이콘'과 링크 블록을 확인한다.

### 2.2 링크 수정 (Update)
- 소유자는 기존에 등록된 URL 주소나 텍스트의 오타를 고치기 위해 해당 링크 영역의 **[수정]** 아이콘을 클릭한다.
- 소유자는 정보를 변경하기 위해 텍스트 입력창에서 텍스트를 수정한 뒤 **[수정 완료]** 버튼을 눌러 변경된 내용을 즉시 확인한다.

### 2.3 링크 삭제 (Delete)
- 소유자는 더 이상 노출하고 싶지 않은 링크를 지우기 위해 해당 링크 블록 우측의 **[삭제]** 아이콘을 누른다.
- 소유자는 실수로 링크를 삭제하는 것을 방지하기 위해 화면에 나타난 **"정말 삭제하시겠습니까?" 재확인 모달(팝업)** 창에서 **[최종 삭제]**를 눌러 안전하게 지운다.
