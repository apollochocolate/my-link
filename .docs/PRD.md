# 📄 마이링크 (MyLink) 기능정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 개발자 및 크리에이터가 자신의 작업물, 포트폴리오, 소셜 미디어 채널 등을 하나의 링크로 모아 공유할 수 있는 '나만의 프로필' 서비스 구축
- **핵심 가치**: 부가적인 꾸밈(테마, 이미지 업로드, 위젯 등) 없이 오로지 '링크 버튼'만을 리스트업해 직관성과 편의성을 극대화한 전면 무료 서비스입니다.

---

## 2. 필수 기능 (MVP - 핵심 요구사항)

### 2.1 회원가입 및 로그인 시스템
- **소셜 로그인 (Google)**: Firebase Authentication을 활용하여 오직 구글 소셜 로그인만 지원합니다.

### 2.2 사용자 프로필 설정 (관리자 뷰)
- **정보 설정**: 닉네임(`displayName`)과 짧은 소개글(`bio`) 입력/수정
- **자유로운 수정**: 닉네임과 소개글은 자유롭게 수정이 가능합니다. (단, 닉네임 수정 시 접속 URL도 함께 변경됩니다.)
- **URL Slug 및 자동 닉네임**: 가입 시 구글 지메일 아이디의 앞부분(예: `abc@gmail.com`의 `abc`)을 추출하여 기본 닉네임(`displayName`)으로 자동 설정합니다. 이 `displayName`이 그대로 개인 접속 URL(Slug)로 사용됩니다. (`예: domain.com/abc`)

### 2.3 링크 리스트 관리 (설정)
- **링크 블록 추가**: 표시할 제목(Title)과 URL을 입력해 새로운 링크 버튼을 추가합니다.
- **파비콘(Favicon) 구글 API 자동 연동**: 별도의 직접 업로드 없이, 구글 파비콘 API(`https://s2.googleusercontent.com/s2/favicons?domain=...`)를 활용해 해당 URL의 파비콘을 자동으로 가져와 아이콘으로 띄웁니다.
- **링크 수정 / 삭제**: 등록된 링크 내용의 수정 및 제거가 가능합니다. (※ 드래그 앤 드롭을 통한 순서 변경, 숨김 토글 등 복잡한 기능은 제외되며 생성된 순서대로 표시됩니다.)

### 2.4 퍼블릭 프로필 페이지 (방문자 뷰)
- **초심플 뷰**: 사용자의 닉네임, 한 줄 소개글, 파비콘이 박힌 심플한 링크 버튼 리스트만 표시되는 모바일/데스크톱 반응형 뷰. (※ 별도 테마 선택 기능 없음)
- **링크 이동**: 버튼 클릭 시 해당 URL로 새 창 띄우기

## 3. 데이터베이스 모델링 (Firebase Firestore)

Firestore의 NoSQL 특성을 살려 **`users` 컬렉션** 하위에 **`links` 서브 컬렉션**을 두는 구조입니다. 이미지 업로드 등의 무거운 처리가 없으므로 가벼운 텍스트 위주의 DB입니다.

### 3.1 `users` (사용자 프로필 컬렉션)
- **Document ID**: `uid` (Firebase Auth UID)
- **Fields**:
  - `email` (String): 계정 연동 이메일 주소
  - `username` (String): 실제 사용자 이름 (본명 등)
  - `displayName` (String): 구글 이메일 아이디 앞부분으로 초기 설정되며, 화면 표시 닉네임이자 URL Slug로 사용됨 (수정 가능)
  - `bio` (String): 프로필에 노출될 짧은 소개글 (수정 가능)
  - `createdAt` (Timestamp): 계정 가입 시기

### 3.2 `users/{uid}/links` (링크 리스트 서브 컬렉션)
- **Document ID**: 자동 생성 ID
- **Fields**:
  - `title` (String): 링크 버튼에 표시할 목적지 이름
  - `url` (String): 실제 이동할 주소
  - `createdAt` (Timestamp): 링크 생성일자 (목록 정렬 기준)
  - *(참고: 파비콘은 DB에 저장하지 않고 클라이언트에서 구글 파비콘 API와 `url`을 결합해 즉시 렌더링하도록 처리)*
